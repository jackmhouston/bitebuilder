## Design Context

### Users
Primary user is a working editor, producer-editor, or post workflow person shaping interview material under deadline pressure. They are typically in the messy middle of story construction: transcript loaded, source XML loaded, trying to find a clean narrative spine fast without losing exact timecode fidelity.

### Brand Personality
Practical, editorial, confident.
The interface should reduce anxiety, reward focus, and feel like a serious craft tool rather than a chatbot demo or generic SaaS dashboard.

### Aesthetic Direction
Selection-first editorial workspace with a warm, paper-and-ink sensibility rather than glossy startup UI. The visual tone should feel closer to an annotated edit board or thoughtful editorial notebook than a futuristic AI panel. Prefer light mode first. Avoid neon, glassmorphism, gradient-heavy AI aesthetics, and generic enterprise dashboard patterns.

### Design Principles
1. Keep transcript, lane, and output visible enough to support real editorial decisions in one surface.
2. Favor legibility, hierarchy, and rhythm over decorative complexity.
3. Make the interface feel like a trusted post-production tool, not an AI toy.
4. Use typography and spacing to create confidence and clarity before adding ornament.
5. Every visual choice should help the editor decide faster.
